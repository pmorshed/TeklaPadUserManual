>Connection status

After running TeklaPad, It'll start searching for a Tekla Structures model. If model be found and everything goes fine, the first tab will be activated and you can choose a template (Fig.2), otherwise this tab remains inactive (Fig.1).

Important Notes:

1- You can retry to establish the connection by clicking on "Refresh"  button.

2- If you faced a screen, as shown in Fig.1, make sure that the Tekla Structure model is open and then click "Refresh".

3- TeklaPad can only connect to single Tekla model. So the connection will be lost if multiple models are running at once.

4- If you find any unsolved/unexpected issues, send us your session log . Find more here

>Manage References and Namespaces:

You can make modifications (Adding / Removing) on extra libraries (or Namespaces) to extend TeklaPad's capabilities.

Hint: It's only for advanced users and can greatly affect the functionality of TeklaPad. So, change the default configurations only if you know what you're doing! 

 

Right-Click > Manage

 

❖References:

 

 

•Adding reference: Click on "Add Reference" > Choose a valid library (*.dll) > Apply

•Removing reference: Right-Click on reference > Remove > Apply

 

ØNote1: After choosing an extra reference, Its most compatible version will be used automatically.

ØNote2: Default references cannot be removed.

 

❖Namespaces:

 

•Adding namespace: Choose namespace from the list > Choose alias (optional) > Click on "Add Namespace" > Apply

•Removing namespace: Right-Click on namespace > Remove > Apply

 

ØNote3: Adding/Removing references may affect available namespaces.

ØNote4: Default namespaces cannot be removed.
