 FAQ

 

Q:

What is TeklaPad?

A:

TeklaPad is an unofficial C# gateway for interacting with Tekla database. It has also a tiny compiler for creating any console application or plug-in for using in Tekla Structures or any other reason.

Q:

Who can use TeklaPad?

A:

TeklaPad contains two independent tabs:

First tab allows you to interact with Tekla database as simple as possible. Basic knowledge about C# and LINQ syntax should be enough for using this tab. Find more here.

Second tab is a tiny compiler which allows you to create console application or plug-in. You should know C# language to use this tab. Find more here.

Note: Don't worry if you are not familiar with C# or Tekla API. This tool will help you practice and increase your knowledge.

Q:

Do i need Visual Studio for using TeklaPad?

A:

No. You don't  need Visual Studio.

Q:

I can not find the GetTeklaReport method in standard API! Why?

A:

TeklaPad contains some additional features which are not available in original API of Tekla Structures. For example "GetTeklaReport" function is not available in original API, whereas it's accessible in first tab of TeklaPad to make workflow more efficient. You can find more explanation here.

Q:

Can i use GetTeklaReport method in second tab?

A:

No. This method (and another additional features) are only available in first tab. Second tab is for coding from the scratch!

Q:

How can i get total (and subtotal) in first tab's data table?

A:

Click on column's header while holding the Control key to get total or subtotal. Find more here.

Q:

How can i highlight objects in model area from first tab's data table?

A:

In order to do this, you must include "GUID" property in your table. Then clicking on the Guid column will result to highlight the object in your model area. Find more in Samples page.

Q:

What about different versions of Tekla Structures?

A:

Don't worry. TeklaPad should works on any version of Tekla Structures that supports API (v.17 and higher).
