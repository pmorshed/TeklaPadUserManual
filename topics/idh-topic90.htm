 FAQ

 

Q:

What is TeklaPad?

A:

TeklaPad is an advanced and user friendly gateway that allows all users (not just advanced users) to interact with Tekla database via API power! It also can be used for creating simple extensions (or plug-ins) for using in daily usage of Tekla Structures.

 

Q:

Who can use TeklaPad?

A:

Anyone who needs the highest control on Tekla Structures or wants to become more professional can use TeklaPad. It contains two MAIN tabs:

First tab allows you to interact with Tekla database (via xAPI) as simple as possible. Basic knowledge about C# and LINQ syntax should be enough for using this tab.

Second tab is a tiny compiler which allows you to create console application or plug-in. You should know C# language to use this tab.

 

Note: Don't worry if you are not familiar with C# or Tekla API. This tool will help you practice and increase your knowledge.

 

Q:

Do i need Visual Studio for using TeklaPad?

A:

No. You don't need Visual Studio.

 

Q:

I can not find the GetTeklaReport  or GetTeklaPullOut  methods in standard API! Why?

A:

TeklaPad contains some additional features (xAPI) which are not available in original API of Tekla Structures. For example "GetTeklaReport" function is not available in original API, whereas it's accessible in first tab of TeklaPad to make workflow more efficient.

 

Q:

Can i use GetTeklaReport  or GetTeklaPullOut  methods in second tab?

A:

No. These methods (and other xAPI features) are only available in first tab. Second tab is for coding from the scratch!

 

Q:

How can i get grand total (and subtotal) in first tab's data table?

A:

There are two ways to do that:

1- "Right-Click" on column's header and choose "Get summation" or "Clear summation".

2- "Click" on column's header while holding the "CTRL" key to get total and subtotal.

Find more here.

 

Q:

How can i highlight objects in model area from first tab's data table?

A:

In order to do this, you must include "GUID" property in your table. Then clicking on the Guid column will result to highlight the object in your model area. Find more in Samples page. You can also use the Highlight feature.

 

Q:

What about different versions of Tekla Structures?

A:

TeklaPad should works on any version of Tekla Structures that supports API (on any machine with .Net framework 4+ installed and sufficient permissions). If you have connectivity problem or run into unexpected issues, send us your session log (Find more here).

 

Q:

What is the difference between "LINQ" and "Standard" syntax in C# ?

A:

There is no any major difference. Both of them can be used in TeklaPad. But LINQ syntax is more readable and is a better choice for queries (especially for newbies). So our general suggestion is that:

Use LINQ syntax for queries and standard syntax for scripts/commands. Here is a simple comparison (in first tab):

 

Simple query (with LINQ syntax):

var MyQuery = from n in Database.OfType<Part>()
              where n.Profile.ProfileString.StartsWith("IPE")
              //Defining required fields:
              let weight = n.GetTeklaReport("weight net")
              let phase = n.GetTeklaReport("phase")
              let color = n.Class
              select new { weight, phase, color };
//Passing the final result to data grid:
return MyQuery;

 

Simple command (with standard syntax):

Model m1 = new Model();
foreach (var item in Database.OfType<Part>())
{
    //Converting part names to uppercase:
    item.Name = item.Name.ToUpper();
    item.Modify();
}
//Updating the model view:
m1.CommitChanges();
//Passing nothing to data grid:
return null;
