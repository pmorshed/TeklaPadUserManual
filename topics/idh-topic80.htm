>About C# syntax

 

Comment lines:

Every line that starts with "//" is a comment line. Comments can be used to document what the program (query/command) does. It makes your query more readable. Since the compiler ignores comments, you can include them anywhere in a program without affecting your code.

 

 

 

Statement terminators:

Every C# statement, not every line, ends with semicolon (";").

 

 

 

 

 

 

>About query & LINQ syntax

Running a query is like asking a question of your Tekla model and returning the result in TeklaPad's data grid.

It doesn't make any changes to the Tekla model. First of all, it is better to know a few points:

 

var keyword:

var (means "variable") is used to define an arbitrary name (for example: SampleQuery) for query (or any variable).

 

Database:

Database is a predefined variable which is responsible for storing the current user selection in Tekla model (Discussed more here)

 

DrawingDatabase:

DrawingDatabase is a predefined variable which is responsible for storing the current user selection in Tekla drawing (Discussed more here)

 

return keyword:

Returns the result to the data table.

Note: If you don't want to pass anything to data table (or if you want to see the console output), you can ignore the return statement.

 

 

 

 

Query body:

C# query syntax (LINQ) is very similar to SQL (Structured Query Language). The LINQ (Language Integrated Query) syntax starts with from keyword and -most of the times- ends with select keyword, conditions are defined by where keyword and field names are defined with let keyword.

 

Hint: using the where and let keywords are arbitrary.

 

 

 

>Look closer to query

In general, creating a query with LINQ syntax contains three basic steps:

1- Defining the original data source (Database/DrawingDatabase).

This step is done by using "from" keyword in LINQ syntax.

       

 

2- Filtering on rows (only choosing some rows by putting some conditions). There are two levels of filtering:

 

2a) "Type" filtering level (Choosing rows by their types):

There are different ways to do but is usually done by using the OfType<T> method.

 

2b) "Property" filtering level (Choosing rows by row properties):

In LINQ syntax this is done by using "where" keyword.
